    Filters

    Simple

    import React from 'react';
import { Table } from 'antd';

const App = () => {
    // Sample data
    const data = [
        { key: '1', name: 'John Doe', age: 32, address: 'New York' },
        { key: '2', name: 'Jane Smith', age: 28, address: 'London' },
        { key: '3', name: 'Sam Brown', age: 22, address: 'Sydney' },
    ];

    // Define columns with filters for name
    const columns = [
        {
            title: 'Name',
            dataIndex: 'name',
            filters: [
                { text: 'John Doe', value: 'John Doe' },
                { text: 'Jane Smith', value: 'Jane Smith' },
            ],
            onFilter: (value, record) => record.name.includes(value),
        },
        {
            title: 'Age',
            dataIndex: 'age',
        },
        {
            title: 'Address',
            dataIndex: 'address',
        },
    ];

    return (
        <Table
            columns={columns}
            dataSource={data}
            pagination={false}
        />
    );
};

export default App;
    1. Data Source: The data array contains records with name, age, and address.
    2. Column Definitions:
      • The name column has filters that allow users to filter by specific names (e.g., "John Doe" or "Jane Smith").
      • The onFilter function checks if the name of the record includes the selected filter value.
    3. Rendering: The Table component displays the columns and data, enabling name-based filtering.

    Age

    import React from 'react';
import { Table } from 'antd';

const App = () => {
    // Sample data
    const data = [
        { key: '1', name: 'John Doe', age: 32, address: 'New York' },
        { key: '2', name: 'Jane Smith', age: 28, address: 'London' },
        { key: '3', name: 'Sam Brown', age: 22, address: 'Sydney' },
    ];

    // Define columns with filters directly
    const columns = [
        {
            title: 'Name',
            dataIndex: 'name',
        },
        {
            title: 'Age',
            dataIndex: 'age',
            filters: [
                { text: 'Under 30', value: 'under30' },
                { text: '30 and above', value: '30plus' },
            ],
            onFilter: (value, record) => {
                if (value === 'under30') return record.age < 30;
                if (value === '30plus') return record.age >= 30;
                return true;
            },
        },
        {
            title: 'Address',
            dataIndex: 'address',
        },
    ];

    return (
        <Table
            columns={columns}
            dataSource={data}
            pagination={false}
        />
    );
};

export default App;
    1. Data Source: The data array contains a list of users.
    2. Column Definitions:
      • The age column includes filters directly defined in the filters property.
      • The onFilter function is specified inline, making it clear and concise.
    3. Rendering: The Table component displays the columns and data, allowing users to filter by age.
    import React from 'react';
import { Table } from 'antd';

const App = () => {
    // Sample data
    const data = [
        { key: '1', name: 'John Doe', age: 32, address: 'New York' },
        { key: '2', name: 'Jane Smith', age: 28, address: 'London' },
        { key: '3', name: 'Sam Brown', age: 22, address: 'Sydney' },
    ];

    // Age filter options
    const ageFilter = [
        { text: 'Under 30', value: 'under30' },
        { text: '30 and above', value: '30plus' },
    ];

    // Filter function for age
    const onFilter = (value, record) => {
        if (value === 'under30') {
            return record.age < 30;
        }
        if (value === '30plus') {
            return record.age >= 30;
        }
        return true;
    };

    // Define columns
    const columns = [
        {
            title: 'Name',
            dataIndex: 'name',
        },
        {
            title: 'Age',
            dataIndex: 'age',
            filters: ageFilter,
            onFilter: onFilter,
        },
        {
            title: 'Address',
            dataIndex: 'address',
        },
    ];

    return (
        <Table
            columns={columns}
            dataSource={data}
            pagination={false}
        />
    );
};

export default App;

    Mastering filters in Ant Design tables

    Mastering filters in Ant Design (antd) tables involves understanding their capabilities and utilizing advanced features for better data management. Here’s a comprehensive guide to help you become proficient with filters in antd tables.

    1. Basic Filtering

    Start with simple filters for specific columns, as shown in previous examples. You can set filters directly in the column definitions and handle filtering logic.

    {
  title: 'Name',
  dataIndex: 'name',
  filters: [
    { text: 'John Doe', value: 'John Doe' },
    { text: 'Jane Smith', value: 'Jane Smith' },
  ],
  onFilter: (value, record) => record.name.includes(value),
}

    2. Multiple Filters

    You can implement multiple filters across different columns. Each column can have its own set of filters, allowing users to refine their searches.

    {
  title: 'Age',
  dataIndex: 'age',
  filters: [
    { text: 'Under 30', value: 'under30' },
    { text: '30 and above', value: '30plus' },
  ],
  onFilter: (value, record) => {
    if (value === 'under30') return record.age < 30;
    if (value === '30plus') return record.age >= 30;
    return true;
  },
}

    3. Custom Filter Logic

    You can create more complex filter logic using custom functions. This is especially useful for non-standard data types or specific business rules.

    const customFilter = (value, record) => {
  // Custom logic based on some criteria
  return record.age === value;
};

{
  title: 'Age',
  dataIndex: 'age',
  filters: [
    { text: '22', value: 22 },
    { text: '32', value: 32 },
  ],
  onFilter: customFilter,
}

    4. Dynamic Filters

    You can generate filter options dynamically based on your data. This ensures users always have relevant options to choose from.

    const uniqueNames = [...new Set(data.map((item) => item.name))];

const columns = [
  {
    title: "Name",
    dataIndex: "name",
    filters: uniqueNames.map((name) => ({ text: name, value: name })),
    onFilter: (value, record) => record.name.includes(value),
  },
];

    5. Search Filtering

    In addition to predefined filters, consider implementing a search input for user-friendly filtering. This allows users to search for specific text within a column.

    import { Input } from "antd";

const App = () => {
  const [searchText, setSearchText] = React.useState("");

  const filteredData = data.filter((item) =>
    item.name.toLowerCase().includes(searchText.toLowerCase())
  );

  return (
    <>
      <Input
        placeholder="Search by name"
        onChange={(e) => setSearchText(e.target.value)}
      />
      <Table dataSource={filteredData} columns={columns} />
    </>
  );
};

    6. Combining Filters

    You can combine multiple filters across different columns, allowing users to filter based on multiple criteria simultaneously.

    const combinedFilter = (filters) => (record) => {
  const matchesName = filters.name ? record.name.includes(filters.name) : true;
  const matchesAge = filters.age ? record.age === filters.age : true;
  return matchesName && matchesAge;
};

    7. Resetting Filters

    Provide a way for users to reset filters easily. This improves usability and allows users to start fresh.

    const resetFilters = () => {
  setFilters({ name: "", age: null });
};

<Button onClick={resetFilters}>Reset Filters</Button>;

    Conclusion

    Mastering filters in antd tables involves understanding how to implement basic filtering, create custom logic, generate dynamic options, and enhance user experience with search functionality. By combining these techniques, you can create powerful and user-friendly data tables in your applications!
