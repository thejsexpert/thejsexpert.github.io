    Application Structure for React.js

    Redux Thunk

    To use an API in a React application with Redux Thunk, you'll need to follow these steps:

    1. Set up Redux Thunk: Ensure you have Redux and Redux Thunk installed and configured in your React application.

    2. Create an API Service: Create a file to manage your API calls, typically using a library like axios or fetch.

    3. Create Action Types: Define the action types for your API calls.

    4. Create Actions: Use Redux Thunk to create asynchronous actions that dispatch the appropriate actions based on the API response.

    5. Create Reducers: Create reducers to handle the actions and update the state accordingly.

    6. Connect Redux to React Components: Use connect or hooks (useSelector and useDispatch) to connect your React components to the Redux store.

    Set up Redux Thunk

    npm install redux react-redux redux-thunk axios

    Folder Structure Explanation

    • actions/: Contains action types and action creators.

      • actionTypes.js: Defines the action types.
      • index.js: Contains action creators using Redux Thunk.

    • api/: Contains API service functions.

      • index.js: Defines the functions to make API calls (e.g., using axios).

    • components/: Contains React components.

      • DataComponent.js: Example component that uses Redux Thunk to fetch and display data.

    • reducers/: Contains Redux reducers.

      • index.js: Combines and exports all the reducers.

    • store/: Contains the Redux store configuration.

      • index.js: Sets up the Redux store with Thunk middleware.

    • App.js: Main app component.

    • index.js: Entry point of the application, where you set up React and Redux.

    • index.css: Global CSS file for styling.

    export const FETCH_DATA_REQUEST = 'FETCH_DATA_REQUEST';
export const FETCH_DATA_SUCCESS = 'FETCH_DATA_SUCCESS';
export const FETCH_DATA_FAILURE = 'FETCH_DATA_FAILURE';

    import {
    FETCH_DATA_REQUEST,
    FETCH_DATA_SUCCESS,
    FETCH_DATA_FAILURE,
} from './actionTypes';
import { fetchData } from '../api';

export const fetchDataRequest = () => ({
    type: FETCH_DATA_REQUEST,
});

export const fetchDataSuccess = (data) => ({
    type: FETCH_DATA_SUCCESS,
    payload: data,
});

export const fetchDataFailure = (error) => ({
    type: FETCH_DATA_FAILURE,
    payload: error,
});

export const fetchDataThunk = () => {
    return (dispatch) => {
        dispatch(fetchDataRequest());
        fetchData()
            .then((response) => {
                dispatch(fetchDataSuccess(response.data));
            })
            .catch((error) => {
                dispatch(fetchDataFailure(error.message));
            });
    };
};

    import axios from 'axios';

const API_URL = 'https://api.example.com';

export const fetchData = () => {
    return axios.get(`${API_URL}/data`);
};

    Folder Structure Explanation

    • api/: Contains files related to API calls.

      • index.js: Manages API requests.

    • components/: Contains reusable React components.

      • DataComponent.js: Example component.

    • redux/: Contains all Redux-related logic.

      • actions/: Contains action types and action creators.
        • actionTypes.js: Defines action types.
        • dataActions.js: Contains action creators related to data.
      • reducers/: Contains reducer functions.
        • dataReducer.js: Reducer for data-related actions.
        • rootReducer.js: Combines all reducers.
      • store/: Contains the Redux store configuration.
        • index.js: Sets up the Redux store with middleware.
      • selectors/: Contains selector functions.
        • dataSelectors.js: Selectors for accessing data state.

    • utils/: Contains utility functions and helpers.

      • apiHelpers.js: Helper functions for API calls.

export const FETCH_USER_SUCCESS = 'FETCH_USER_SUCCESS';
export const FETCH_USER_FAILURE = 'FETCH_USER_FAILURE';
export const CREATE_USER_REQUEST = 'CREATE_USER_REQUEST';
export const CREATE_USER_SUCCESS = 'CREATE_USER_SUCCESS';
export const CREATE_USER_FAILURE = 'CREATE_USER_FAILURE';
// Other action types...

    import {
    FETCH_USER_REQUEST,
    FETCH_USER_SUCCESS,
    FETCH_USER_FAILURE,
    CREATE_USER_REQUEST,
    CREATE_USER_SUCCESS,
    CREATE_USER_FAILURE,
} from './actionTypes';

import {
    getUserById,
    createUser,
    updateUser,
    deleteUser,
    searchUsers,
} from '../../api/user';

import { normalizeUserData } from '../utils/normalize';

// Fetch user by ID
export const fetchUser = (id) => {
    return async (dispatch) => {
        dispatch({ type: FETCH_USER_REQUEST });

        try {
            const response = await getUserById(id);
            const normalizedData = normalizeUserData(response.data);
            dispatch({ type: FETCH_USER_SUCCESS, payload: normalizedData });
        } catch (error) {
            dispatch({ type: FETCH_USER_FAILURE, payload: error.message });
        }
    };
};

// Create user
export const addUser = (userData) => {
    return async (dispatch) => {
        dispatch({ type: CREATE_USER_REQUEST });

        try {
            const response = await createUser(userData);
            const normalizedData = normalizeUserData(response.data);
            dispatch({ type: CREATE_USER_SUCCESS, payload: normalizedData });
        } catch (error) {
            dispatch({ type: CREATE_USER_FAILURE, payload: error.message });
        }
    };
};

// Update user
export const modifyUser = (id, userData) => {
    return async (dispatch) => {
        dispatch({ type: UPDATE_USER_REQUEST });

        try {
            const response = await updateUser(id, userData);
            const normalizedData = normalizeUserData(response.data);
            dispatch({ type: UPDATE_USER_SUCCESS, payload: normalizedData });
        } catch (error) {
            dispatch({ type: UPDATE_USER_FAILURE, payload: error.message });
        }
    };
};

// Delete user
export const removeUser = (id) => {
    return async (dispatch) => {
        dispatch({ type: DELETE_USER_REQUEST });

        try {
            await deleteUser(id);
            dispatch({ type: DELETE_USER_SUCCESS, payload: id });
        } catch (error) {
            dispatch({ type: DELETE_USER_FAILURE, payload: error.message });
        }
    };
};

// Search users
export const findUsers = (queryParams) => {
    return async (dispatch) => {
        dispatch({ type: SEARCH_USERS_REQUEST });

        try {
            const response = await searchUsers(queryParams);
            const normalizedData = normalizeUserData(response.data);
            dispatch({ type: SEARCH_USERS_SUCCESS, payload: normalizedData });
        } catch (error) {
            dispatch({ type: SEARCH_USERS_FAILURE, payload: error.message });
        }
    };
};

    Reducers

    redux/reducers/

    This reducer handles the normalization of entities, which helps in managing relationships between different entities.

    import { combineReducers } from 'redux';
import {
    FETCH_USER_SUCCESS,
    CREATE_USER_SUCCESS,
    UPDATE_USER_SUCCESS,
    SEARCH_USERS_SUCCESS,
} from '../actions/actionTypes';

const initialState = {
    users: {},
};

const users = (state = initialState.users, action) => {
    switch (action.type) {
        case FETCH_USER_SUCCESS:
        case CREATE_USER_SUCCESS:
        case UPDATE_USER_SUCCESS:
        case SEARCH_USERS_SUCCESS:
            return {
                ...state,
                ...action.payload.entities.users,
            };
        default:
            return state;
    }
};

export default combineReducers({
    users,
});

    This reducer handles the specific logic related to users, such as loading states and errors.

    import {
    FETCH_USER_REQUEST,
    FETCH_USER_SUCCESS,
    FETCH_USER_FAILURE,
    CREATE_USER_REQUEST,
    CREATE_USER_SUCCESS,
    CREATE_USER_FAILURE,
    UPDATE_USER_REQUEST,
    UPDATE_USER_SUCCESS,
    UPDATE_USER_FAILURE,
    DELETE_USER_REQUEST,
    DELETE_USER_SUCCESS,
    DELETE_USER_FAILURE,
    SEARCH_USERS_REQUEST,
    SEARCH_USERS_SUCCESS,
    SEARCH_USERS_FAILURE,
} from '../actions/actionTypes';

const initialState = {
    loading: false,
    error: null,
    allIds: [],
    currentUser: null,
};

const usersReducer = (state = initialState, action) => {
    switch (action.type) {
        case FETCH_USER_REQUEST:
        case CREATE_USER_REQUEST:
        case UPDATE_USER_REQUEST:
        case DELETE_USER_REQUEST:
        case SEARCH_USERS_REQUEST:
            return {
                ...state,
                loading: true,
                error: null,
            };
        case FETCH_USER_SUCCESS:
            return {
                ...state,
                loading: false,
                currentUser: action.payload.result,
            };
        case CREATE_USER_SUCCESS:
            return {
                ...state,
                loading: false,
                allIds: [...state.allIds, action.payload.result],
            };
        case UPDATE_USER_SUCCESS:
            return {
                ...state,
                loading: false,
            };
        case DELETE_USER_SUCCESS:
            return {
                ...state,
                loading: false,
                allIds: state.allIds.filter((id) => id !== action.payload),
            };
        case SEARCH_USERS_SUCCESS:
            return {
                ...state,
                loading: false,
                allIds: action.payload.result,
            };
        case FETCH_USER_FAILURE:
        case CREATE_USER_FAILURE:
        case UPDATE_USER_FAILURE:
        case DELETE_USER_FAILURE:
        case SEARCH_USERS_FAILURE:
            return {
                ...state,
                loading: false,
                error: action.payload,
            };
        default:
            return state;
    }
};

export default usersReducer;

    Combine the entity and user-specific reducers to create the root reducer.

    import { combineReducers } from 'redux';
import entitiesReducer from './entitiesReducer';
import usersReducer from './usersReducer';

const rootReducer = combineReducers({
    entities: entitiesReducer,
    users: usersReducer,
});

export default rootReducer;

    Normalization

    Normalize the data received from the API to ensure it is stored in a flat structure, making it easier to manage and update.

    redux/utils/normalize.js 
    import { normalize, schema } from 'normalizr';

// Define a user schema
const user = new schema.Entity('users');

// Define a function to normalize user data
export const normalizeUserData = (data) => {
    return normalize(data, user);
};

    Selectors

    Use selectors to access and derive data from the state, keeping your components clean and focused on rendering.

    redux/selectors/userSelectors.js 
    import { createSelector } from 'reselect';

export const getUsersState = (state) => state.users;
export const getEntitiesState = (state) => state.entities;

export const getAllUserIds = createSelector(
    [getUsersState],
    (usersState) => usersState.allIds
);

export const getUserById = (state, userId) =>
getEntitiesState(state).users[userId];

export const getAllUsers = createSelector(
    [getAllUserIds, getEntitiesState],
    (allIds, entities) => allIds.map((id) => entities.users[id])
);

export const getCurrentUser = createSelector(
    [getUsersState, getEntitiesState],
    (usersState, entities) => entities.users[usersState.currentUser]
);

    Connecting Components

    components/UserComponent.js 
    import React, { useEffect } from 'react';
import { useSelector, useDispatch } from 'react-redux';
import { fetchUser, addUser, modifyUser, removeUser, findUsers } from '../redux/actions/userActions';
import { getAllUsers, getCurrentUser, getUserLoading, getUserError } from '../redux/selectors/userSelectors';

const UserComponent = () => {
    const dispatch = useDispatch();
    const users = useSelector(getAllUsers);
    const currentUser = useSelector(getCurrentUser);
    const loading = useSelector(getUserLoading);
    const error = useSelector(getUserError);

    useEffect(() => {
        dispatch(fetchUser(1)); // Fetch user with ID 1
    }, [dispatch]);

    const handleAddUser = (userData) => {
        dispatch(addUser(userData));
    };

    const handleUpdateUser = (id, userData) => {
        dispatch(modifyUser(id, userData));
    };

    const handleDeleteUser = (id) => {
        dispatch(removeUser(id));
    };

    const handleSearchUsers = (queryParams) => {
        dispatch(findUsers(queryParams));
    };

    return (
        <div>
            {loading ? (
                <p>Loading...</p>
            ) : error ? (
                <p>Error: {error}</p>
            ) : (
                <div>
                <h1>User Details</h1>
                {currentUser && <div>{currentUser.name}</div>}
                <h1>All Users</h1>
                <ul>
                    {users.map((u) => (
                    <li key={u.id}>{u.name}</li>
                    ))}
                </ul>
                </div>
            )}
            <button onClick={() => handleAddUser({ name: 'New User' })}>Add User</button>
            <button onClick={() => handleUpdateUser(1, { name: 'Updated User' })}>Update User</button>
            <button onClick={() => handleDeleteUser(1)}>Delete User</button>
            <button onClick={() => handleSearchUsers({ name: 'John' })}>Search Users</button>
        </div>
    );
};

export default UserComponent;

    This structure allows for scalability, as each part of the state is managed independently and can be easily extended with new entities or additional logic. The use of selectors ensures that the component logic remains clean and focused on rendering, while the normalization of data helps manage relationships and avoid nested state.
