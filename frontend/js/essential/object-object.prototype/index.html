    Object & Object.Prototype

    The difference between Object and Object.prototype in JavaScript lies in their roles and usage within the language. Here's an in-depth explanation:

    1. Object

    • Definition: Object is a built-in global constructor function in JavaScript.
    • Purpose: It is used to create and manage objects, and it also provides utility methods for working with objects.
    Key Characteristics
    • It's a function that serves as the parent of all objects in JavaScript.
    • Provides static methods like Object.keys(), Object.values(), Object.entries(), etc.
    • You can use it to create objects explicitly: new Object() (though Object.create() or object literals {} are more common).
    // Using Object as a constructor
const obj1 = new Object();
obj1.name = "Alice";
console.log(obj1); // { name: 'Alice' }

// Using static methods provided by Object
const obj2 = { a: 1, b: 2 };
console.log(Object.keys(obj2)); // ['a', 'b']
console.log(Object.values(obj2)); // [1, 2]
console.log(Object.entries(obj2)); // [['a', 1], ['b', 2]]

    2. Object.prototype

    • Definition: Object.prototype is the prototype object from which all JavaScript objects inherit properties and methods.
    • Purpose: It acts as the base object, and every object created in JavaScript implicitly links to Object.prototype (or a prototype that inherits from it).
    Key Characteristics
    • Contains instance methods like .toString(), .hasOwnProperty(), .isPrototypeOf(), etc.
    • When you create an object, its prototype chain ultimately leads to Object.prototype.
    • It defines the default behavior for all objects unless overridden.
    const obj3 = { a: 1 };

// Accessing methods from Object.prototype
console.log(obj3.hasOwnProperty("a")); // true
console.log(obj3.toString()); // [object Object]

// Adding a custom method to Object.prototype
Object.prototype.greet = function () {
  return "Hello from Object.prototype!";
};
console.log(obj3.greet()); // "Hello from Object.prototype!"

// Caution: Modifying Object.prototype affects all objects

    3. Key Differences Between Object and Object.prototype

    Feature Object Object.prototype
    Type Constructor function Prototype object
    Purpose Used to create objects and access static methods Provides default methods and properties to all objects
    Access Static methods like Object.keys() Instance methods like .toString()
    Inheritance Does not inherit from anything Root of the prototype chain
    Extensibility Adding properties or methods is uncommon Can extend to modify behavior of all objects
    Example Methods Object.keys(), Object.create() .hasOwnProperty(), .toString()

    4. How They Work Together

    When you create an object, it inherits from Object.prototype. For example:

    const myObj = { a: 1 };

// Prototype chain
console.log(Object.getPrototypeOf(myObj) === Object.prototype); // true

       └─> myObj
    • Object.prototype is the prototype of all plain objects.
    • Object is the global constructor function that creates objects and provides static methods.

    Array

    An array is an ordered list of values typically created with the intention of looping through numerically indexed values, beginning with the index zero.

    What you need to know is that arrays are numerically ordered sets, as opposed to objects which have property names associated with values in non-numeric order.

    Essentially, arrays use numbers as a lookup key, while objects have user-defined property names. JavaScript does not have true associative arrays, but objects can be used to achieve the functionality of associative arrays.

    var myArray = ["blue", "green", "orange", "red"];
console.log(myArray[0]); // Logs blue using the 0 index to access the string in myArray.

// Versus

var myObject = {
  // aka an associative array/hash, known as an object in JavaScript.
  blue: "blue",
  green: "green",
  orange: "orange",
  red: "red",
};
    Note

    Arrays can hold any type of values, and these values can be updated or deleted at any time.

    If you need a hash (aka associative array), an object is the closest solution.

    An Array() is just a special type of Object(). That is, Array() instances are basically Object() instances with a couple of extra functions (e.g., .length and a built-in numeric index).

    Values contained in an array are commonly referred to as elements.
