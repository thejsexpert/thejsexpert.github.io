    Installation
    npm install -g pnpm
    Scaffolding Your First Vite Project
    pnpm create vite

    # Install pnpm globally if not installed
npm install -g pnpm

# Create the project with React and TypeScript template
pnpm create vite my-react-ts-app --template react-ts

# Navigate into the project directory
cd my-react-ts-app

# Install dependencies
pnpm install

# Start the development server
pnpm run dev

# Build the production version of the app
pnpm run build

# Preview the production build
pnpm run preview
    1. npm list Shows Local Dependencies, Not Global

    The npm list command by default shows the dependencies for your current project (i.e., the local node_modules in the current directory). If you installed pnpm globally, it will not show up in the output of npm list unless you specify the global flag.

    Solution:

    To list globally installed packages with npm, use the following command:

    npm list -g --depth=0

    This will show all globally installed packages (including pnpm) at the top level.

    PNPM: A Comprehensive Guide

    PNPM (pronounced as "Pee-En-Pee-Em") is a fast, disk space-efficient package manager for JavaScript, specifically designed to handle the installation of dependencies for Node.js projects. It is an alternative to other package managers like npm (Node Package Manager) and Yarn, offering better performance and disk space utilization.

    Let's break it down step by step, from basic concepts to advanced usage.

    Tip
    • Package Manager: Like npm and Yarn, PNPM is a tool that helps you manage libraries and dependencies in your JavaScript or Node.js projects.
    • Efficiency: What sets PNPM apart is its focus on optimizing disk space usage and speeding up dependency installation, especially in large projects.
    • Global Store: Instead of downloading a package separately for each project, PNPM uses a global store where it keeps downloaded packages. It then links them to each project, minimizing duplication and reducing disk usage.

    1. Performance:

      PNPM installs packages faster than npm and Yarn, especially in large projects. This is because it stores packages globally, avoiding redundant downloads.

    2. Disk Space Efficiency:

      PNPM uses hard links to share common dependencies between different projects. If two projects use the same package (with the same version), PNPM wont download it twice. Instead, it creates a link to the existing package in the global store, which saves disk space.

    3. Consistency:

      PNPM strictly enforces a deterministic node_modules structure. This means the dependency tree is always the same, even if you install packages on different machines. This can help avoid "it works on my machine" issues.

    4. Better Dependency Resolution:

      PNPM uses a strict algorithm for resolving dependencies, meaning that if a package needs a specific version of a dependency, it will only install that version (no more "hoisting" to higher levels of the dependency tree, which can cause issues).

    • Performance and Efficiency Needs:

      If youre working on a large project or managing multiple projects that share dependencies, PNPM can dramatically reduce installation times and disk usage.

    • Modern JavaScript Ecosystem:

      For frontend and backend projects, especially when you're managing monorepos or large-scale applications, PNPMs optimization features come in handy.

    • If You're Tired of NPM's Performance and Dependency Issues:

      If you've encountered slow npm install times or inconsistencies in dependency resolutions (e.g., issues caused by hoisting in node_modules), PNPM can solve many of these problems.

    1. Legacy Projects:

      If you're working on a project that requires using npm or Yarn due to compatibility or organizational constraints, it might not be worth switching to PNPM unless there's a clear need for the benefits it offers.

    2. Third-Party Tools/Plugins:

      Some tools and integrations are still optimized for npm or Yarn. In rare cases, certain third-party tools might not support PNPM well, though this is becoming less of an issue over time.

    The major benefit of using pnpm over npm is its efficient disk space usage and performance optimization, particularly when dealing with large projects or monorepos.

    Difference between install and add in pnpm

    In pnpm, both add and install are commands related to managing dependencies in a project, but they serve different purposes:

    The pnpm add command is used to add a new dependency to your project. This will install the dependency and update the pnpm-lock.yaml and package.json files accordingly.

    • Usage:

      • pnpm add <package> Adds a package as a regular dependency.
      • pnpm add <package>@<version> Adds a specific version of a package.
      • pnpm add <package> --dev Adds a package as a development dependency.
      • pnpm add <package> --save-peer Adds a package as a peer dependency.

    • Example:

      pnpm add lodash

      This will add lodash as a dependency to your package.json and install it into the node_modules folder.

    The pnpm install command is used to install all dependencies listed in the package.json file. It will install both regular and development dependencies that are already specified in your project, without adding or modifying the package.json.

    • Usage:

      • pnpm install Installs all dependencies in the current project.
      • pnpm install --prod Installs only production dependencies (ignores dev dependencies).
      • pnpm install <package> Installs a specific package, if you want to install without adding it to the package.json (essentially a one-off installation).

    • Example:

      pnpm install

      This will install all dependencies defined in your package.json into the node_modules folder.

    • pnpm add: Used to add a new dependency (and update package.json), and install it.
    • pnpm install: Used to install the dependencies already defined in package.json, including any changes to the lockfile.

    • In summary:

      • Use pnpm add when you want to add a new package to the project.
      • Use pnpm install when you want to install all the dependencies that are already listed in package.json.

    How to Use PNPM

    Example

    You can install PNPM globally using npm or Yarn

    • Via npm:

      npm install -g pnpm

    • Via Yarn:

      npm install -g pnpm

    After installation, you can check the version to confirm it worked:

    pnpm --version

    • Initialize a new project (package.json): To start using PNPM in a new project, run the following:

      pnpm init

      This will create a package.json file, just like npm init.

    • Install Dependencies: To install all dependencies listed in the package.json file:

      pnpm install

      PNPM will install dependencies into the node_modules folder using hard links from the global store.

    • Install a Package (One-time install): To install a specific package and add it to dependencies in package.json

      pnpm add <package-name>

      You can also install development dependencies by using -D:

      pnpm add <package-name> -D

    • Install a Specific Version of a Package:

      pnpm add <package-name>@<version>

      For example:

      pnpm add lodash@4.17.21

    • Uninstall a Package: To remove a package:

      pnpm remove <package-name>

    • Workspaces / Monorepos: PNPM supports workspaces, which is useful when you're managing multiple packages in a single repository (a monorepo). You can set up multiple packages and install them all with one command:

      1. Create a pnpm-workspace.yaml file in the root directory.

      2. List your packages in that file:

        packages:
    - 'packages/*'

      3. Inside your packages, use pnpm install to install dependencies across the entire workspace.

      PNPM automatically links dependencies between your packages within the monorepo, which is a big advantage when working with large codebases that share dependencies.

    • Run Scripts: You can use PNPM to run scripts defined in your package.json:

      pnpm run <script-name>

    • Global Packages: To install packages globally using PNPM, use:

      pnpm add -g <package-name>

      This will install the package globally, and you can run it directly from the command line.

    • Auditing Dependencies: To check for security vulnerabilities in your projects dependencies:

      pnpm audit

    • Linking Local Packages: If you're developing multiple local packages and need to link them, you can use the pnpm link command to create symlinks between them:

      pnpm link <package-name>

    • Custom Store: You can customize where PNPM stores its global store. For example, you can set a custom directory by adding this to your .npmrc or pnpmfile.js:

      pnpm store-dir /path/to/store

    Comparing PNPM with NPM and Yarn

    Feature PNPM NPM Yarn
    Speed Fast (due to global store and hard links) Slower (redownloads dependencies for every project) Faster than npm, but not as fast as PNPM
    Disk Usage Low (global store, hard links) High (duplicates dependencies across projects) Lower than npm but higher than PNPM
    Deterministic Install Yes (strict dependency tree) No (hoisting can cause issues) Yes
    Workspaces Support Yes No (requires third-party tool) Yes
    CLI Commands Similar to npm Standard npm commands Similar to npm
    Community Growing rapidly Largest (since it's the default package manager for Node.js) Large, but smaller than npm

    Conclusion

    PNPM is a fast, efficient, and modern alternative to npm and Yarn, offering unique advantages in performance, disk space usage, and dependency management. It's especially beneficial for large-scale projects, monorepos, or teams looking for a more efficient way of managing Node.js dependencies.

    When adopting PNPM, the transition is straightforward if you're familiar with npm or Yarn. The key advantages faster installs, less disk usage, and strict dependency resolution can be highly beneficial for large projects, while its simplicity and ease of use ensure its a good fit for individual developers as well.

    vite overview

    Vite, being a fast and flexible build tool, has a rich ecosystem of plugins that extend its functionality for various tasks such as code transformation, asset handling, testing, and more. Below are some of the commonly used types of plugins in the Vite ecosystem:

    1. Official Vite Plugins

    These plugins are developed and maintained by the Vite team itself and are included in the core ecosystem:

    • @vitejs/plugin-vue Provides support for Vue 3 in Vite. This plugin handles .vue file parsing, component support, and more.

    • @vitejs/plugin-react Adds React Fast Refresh and support for React files (.jsx/.tsx).

    • @vitejs/plugin-vue-jsx Adds support for Vue 3 JSX syntax.

    • @vitejs/plugin-legacy Provides support for legacy browsers by generating polyfills and transforming code accordingly.

    • vite-plugin-svelte Adds support for Svelte, enabling the compilation of .svelte files.

    • vite-plugin-static-copy A plugin that allows you to copy static assets (files or directories) into the build output.

    2. CSS-related Plugins

    • vite-plugin-sass-dts Generates .d.ts TypeScript declaration files for SASS variables, mixins, and functions.

    • vite-plugin-postcss Allows custom PostCSS configuration for your project.

    • vite-plugin-sass Enhances Vite's Sass/SCSS support.

    • vite-plugin-tailwind Optimizes Tailwind CSS configuration and integration.

    • vite-plugin-stylus Adds Stylus support for Vite projects.

    3. TypeScript/JSX/TSX-related Plugins

    • vite-plugin-checker Provides TypeScript type checking and ESLint checking during development.

    • vite-plugin-tsconfig-paths Automatically resolves TypeScript paths based on the tsconfig.json paths configuration.

    • vite-plugin-swc A plugin that uses swc (Speedy Web Compiler) to transpile JavaScript and TypeScript instead of Babel.

    • vite-plugin-prettier Integrates Prettier into the Vite build process to format files automatically.

    4. Development Tools & Features

    • vite-plugin-env-compatible Provides compatibility for environment variables in Vite, ensuring that .env files and variables are correctly handled.

    • vite-plugin-serve Serves static files during development or production using Vites dev server.

    • vite-plugin-progress Displays a progress bar in the terminal during the build process.

    • vite-plugin-mock Allows for easy integration of mock APIs during development.

    • vite-plugin-html Extends HTML handling in Vite with various enhancements, including template injections.

    5. Code Splitting & Optimization Plugins

    • vite-plugin-prerender Allows you to prerender specific pages during the build process, often used for static site generation.

    • vite-plugin-import Optimizes imports and reduces bundle size by automatically removing unused code or assets from dependencies.

    • vite-plugin-purgecss Integrates PurgeCSS with Vite to remove unused CSS and reduce the final bundle size.

    • vite-plugin-bundle-visualizer Visualizes your final bundle with detailed stats about how much space different dependencies take up.

    6. Image & Asset-related Plugins

    • vite-plugin-imagemin Optimizes images in your project to reduce their size without losing quality.

    • vite-plugin-assets Handles static assets, including images, fonts, etc., and can bundle them efficiently in the final build.

    • vite-plugin-optimize-images Similar to the imagemin plugin but with additional features to optimize the handling of images.

    7. Backend & API-related Plugins

    • vite-plugin-graphql Provides GraphQL-related tooling, such as schema generation and query handling.

    • vite-plugin-cloudflare-workers Allows you to deploy your Vite project as a Cloudflare Worker.

    • vite-plugin-netlify Configures your Vite project for optimal deployment on Netlify.

    8. Testing Plugins

    • vite-plugin-test A plugin for running unit tests, integration tests, or end-to-end tests with Vite.

    • vite-plugin-jest Integrates Jest testing framework into Vite projects for unit testing.

    • vite-plugin-vitest Adds Vitest, a fast unit test runner built for Vite, to your project.

    9. Framework-Specific Plugins

    • vite-plugin-nuxt Adds support for Nuxt.js features in Vite.

    • vite-plugin-next Adds support for Next.js features, such as React Server Components, in Vite-based projects.

    • vite-plugin-markdown Processes Markdown files and allows importing them into your project as .md files.

    10. Miscellaneous Plugins

    • vite-plugin-svgr This plugin enables importing SVG files as React components, which is especially useful for projects with heavy SVG use.

    • vite-plugin-mdx Adds support for MDX (Markdown with embedded JSX) in Vite.

    • vite-plugin-webfont-dl Automatically downloads and serves web fonts during the build process.

    • vite-plugin-eslint Automatically runs ESLint in the development environment to lint files on save.

    11. Custom Plugins

    • vite-plugin-custom If you're developing a Vite plugin for your specific use case, you can write your own custom plugin by using the Vite plugin API, which allows you to hook into Vites lifecycle events.

    Conclusion

    The Vite plugin ecosystem is vast and continues to grow. By selecting the right plugins for your project, you can tailor Vite to suit your specific needs, whether it's optimizing your build, adding support for additional frameworks, or improving your development experience.

    If youre looking for specific plugins for your project, the Vite Plugin Catalog is a great place to start!

    Summary of Must-Have Plugins for Your Setup React:
    • @vitejs/plugin-react Essential for React projects.
    • vite-plugin-checker TypeScript and ESLint integration during development.
    • vite-plugin-tsconfig-paths Ensures TypeScript path aliases work seamlessly in Vite.
    • vite-plugin-swc (Optional for speed) Uses swc to speed up TypeScript compilation.
    • vite-plugin-svgr (Optional) If using SVGs as React components.
    • vite-plugin-eslint (Optional) For continuous linting during development.
