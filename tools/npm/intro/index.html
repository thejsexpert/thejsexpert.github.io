    Intro

    Overview

    • npm (Node Package Manager)

      Comes pre-installed with Node.js, no additional setup required.

      Reference

    • Yarn

      Excellent support for monorepos and managing multiple packages within a single repository.

      Reference

    • pnpm (Performant npm)

      Uses a unique approach of symlinking packages which saves space and speeds up installations.

      Reference

    Decision Factors

    1. Performance Needs: If speed and efficiency are crucial, consider Yarn or pnpm.
    2. Monorepo Management: For projects with multiple packages, Yarn or pnpm’s workspace features are beneficial.
    3. Team Familiarity: Choose the package manager that your team is most comfortable with to minimize the learning curve.
    4. Ecosystem and Tooling: Consider the compatibility of the package manager with other tools and services you use.

    Summary

    • Use npm if you prefer the default, out-of-the-box solution with Node.js.
    • Use Yarn if you need faster installs, stability, and good monorepo support.
    • Use pnpm if you want the best performance and disk space efficiency, and are open to adopting newer tools.

    npm

    ninja@ninja-OptiPlex-3080:/$ npm list
/
└── (empty)

ninja@ninja-OptiPlex-3080:/$ npm list --depth=0
/
└── (empty)

ninja@ninja-OptiPlex-3080:/$ nx --version
Nx Version:
- Local: Not found
- Global: v20.3.3

ninja@ninja-OptiPlex-3080:/$ npm list -g --depth=0

/home/ninja/.nvm/versions/node/v20.17.0/lib
├── corepack@0.29.3
├── npm@10.8.2
├── nx@20.3.3
└── pnpm@9.14.2
    To clarify
    • Local packages (in a specific project folder) can be listed with npm list.
    • Global packages are stored separately from the local project and need to be listed with npm list -g.
    To check global packages

    You can use this command to list all globally installed packages:

    npm list -g --depth=0
    • npm list --depth=0: This command shows the top-level dependencies in the current directory, but it only shows packages listed in the node_modules of the current working directory. Since you ran it at the root directory /, which likely doesn't have a node_modules folder or a local package.json file, it shows as empty.
