    JS CONSTANT

    Naming Conventions

    The naming conventions for constant files and their values inside generally follow best practices for clarity, readability, and consistency in the codebase. Here's a recommended approach to naming constants, particularly in JavaScript and React applications:

    1. File Naming Convention

    • Use Uppercase for Constant Files: Constant files are typically named in all uppercase letters, often using underscores to separate words (snake_case). This makes it clear that the file contains constant values that should not change.

      • Example: FOREX_DEAL.js, CURRENCY_CONSTANTS.js, FORM_CONSTANTS.js

    • Avoid Special Characters: Stick with letters, numbers, and underscores (_) to make the filenames easily readable and compatible with different systems.

    • Group Constants Based on Context: If your project has multiple areas with constants, consider organizing them into different files based on their relevance (e.g., one file for form constants, one for API URLs, etc.).

    Example
    • USER_CONSTANTS.js for user-related constants
    • API_URLS.js for API endpoint constants
    • FORM_CONSTANTS.js for form-related constants like your FOREX_DEAL

    2. Constant Naming Convention Inside the File

    • Uppercase Snake Case for Constants: Constants are often written in all uppercase letters with underscores separating words. This clearly distinguishes constants from variables and regular functions.

      • Example: BUYING_CURRENCY, SELLING_CURRENCY, COUNTERPARTY

    • Descriptive and Meaningful Names: Constant names should describe what the constant represents. Use clear, self-explanatory names to improve readability.

      • Instead of generic names like VALUE1 or KEY_A, use something more meaningful, like MAX_RETRY_ATTEMPTS or DEFAULT_LANGUAGE.

    • Group Related Constants: For objects or grouped values, use camel case for keys and avoid adding "constants" to the variable names as its implied.

    Example

    const FOREX_DEAL = { ... } instead of const FOREX_DEAL_CONSTANTS = { ... }

    • Booleans: For boolean constants, use is, has, can, should, etc., as prefixes to make their purpose clearer.

      • Example: IS_ENABLED, HAS_PERMISSION

    3. Example of Constant File: FOREX_DEAL.js

    // FOREX_DEAL.js

export const FOREX_DEAL = {
  BUYING_CURRENCY: "", // Currency that is being bought
  SELLING_CURRENCY: "", // Currency that is being sold
  COUNTERPARTY: "", // The party you're dealing with
  COUNTERPARTY_DEALER: "", // The specific dealer in the counterparty
  BUYING_AMOUNT: "", // Amount of buying currency
  RATE: "", // Exchange rate
  SELLING_AMOUNT: "", // Amount of selling currency
  WE_PAY_FORM: "", // Payment form used
  THEY_RECEIVED_AT: "", // Time at which they received the amount
  WE_RECEIVED_AT: "", // Time at which we received the amount
  APPROVAL_PERSON: "", // Person who approves the deal
  REMARKS: "", // Additional remarks
};

    4. Key Practices for Constants:

    • Do Not Use Mutable Values: Constants should never hold mutable values, like arrays or objects that can be modified unless they're explicitly meant to store unchanging data.

    • Avoid Magic Numbers: Constants are useful for representing values that would otherwise be magic numbers or strings in your code (e.g., 3, 45, true). Assign them a meaningful name.

    • Example: Instead of if (status === 1), use const STATUS_ACTIVE = 1; and then if (status === STATUS_ACTIVE).

    • Document Complex Constants: If you have complex constants, especially objects, consider adding comments to explain their purpose, especially when the structure is not self-evident.

    5. Example File and Structure in a React Project

    Heres a directory structure for an organized set of constants in a React project:

    /src
  /constants
    FOREX_DEAL.js           // Forex related constants (e.g., initial form values)
    CURRENCY_CONSTANTS.js   // Constants for different currencies
    API_URLS.js             // API URLs for making network requests
    ERROR_MESSAGES.js       // Constants for error messages
    FORM_CONSTANTS.js       // General form constants (e.g., validation rules)

    Each of these files would define constants that are specific to their respective context. For example:

    // CURRENCY_CONSTANTS.js
export const CURRENCY_TYPES = {
  USD: "USD",
  EUR: "EUR",
  GBP: "GBP",
  JPY: "JPY",
};

    By using these practices, youll maintain a clean, organized codebase that makes it easy for anyone (including future you) to understand the structure and purpose of your constants.

    Using CONSTANTS

    To use constants defined in another file, you'll need to import them properly into the file where you're using them. Here's how you can do that:

    1. Create a Constants File (e.g., FOREX_DEAL.js)

    First, create a file where you define your constants, such as FOREX_DEAL.js.

    // FOREX_DEAL.js

export const FOREX_DEAL = {
  BUYING_CURRENCY: "buyingCurrency", // String value of the field name
  SELLING_CURRENCY: "sellingCurrency",
  COUNTERPARTY: "counterparty",
  COUNTERPARTY_DEALER: "counterPartyDealer",
  BUYING_AMOUNT: "buyingAmount",
  RATE: "rate",
  SELLING_AMOUNT: "sellingAmount",
  WE_PAY_FORM: "wePayForm",
  THEY_RECEIVED_AT: "theyReceivedAt",
  WE_RECEIVED_AT: "weReceivedAt",
  APPROVAL_PERSON: "approvalPerson",
  REMARKS: "remarks",
};

    2. Import the Constants into Your Component File

    Next, import the FOREX_DEAL constant into the component file where you're using the useEffect.

    import React, { useEffect } from "react";
import { Form } from "antd";
import { FOREX_DEAL } from "./FOREX_DEAL"; // Import the constants

const YourComponent = () => {
  const [form] = Form.useForm();

  useEffect(() => {
    // Get the value of the buyingCurrency field
    let buyingCurrency = form.getFieldValue(FOREX_DEAL.BUYING_CURRENCY);

    if (buyingCurrency) {
      // Set WE_RECEIVED_AT and THEY_RECEIVED_AT fields to null
      form.setFieldsValue({
        [FOREX_DEAL.WE_RECEIVED_AT]: null,
        [FOREX_DEAL.THEY_RECEIVED_AT]: null,
      });

      // Assuming deleteFromItem is a utility to remove the field, it should be called with the field names
      deleteFromItem(FOREX_DEAL.WE_RECEIVED_AT);
      deleteFromItem(FOREX_DEAL.THEY_RECEIVED_AT);
    }
  }, [form.getFieldValue(FOREX_DEAL.BUYING_CURRENCY)]); // Trigger the effect when buyingCurrency changes

  return <Form form={form}>{/* Form Items go here */}</Form>;
};

export default YourComponent;
    Key Points:

    1. Import the Constants:

    • Use the import statement to bring in the constants defined in another file (FOREX_DEAL in this case).
    • In the useEffect, we now reference FOREX_DEAL.BUYING_CURRENCY, FOREX_DEAL.WE_RECEIVED_AT, and other constants instead of hardcoding the field names.

    2. Use Constants in Dynamic Field References:

    • You can use constants to dynamically access or set field values in the form.setFieldsValue and form.getFieldValue.
    • When you set or get a field value, use the bracket notation ([]) to reference the field names dynamically from the constants object.
    Benefits of This Approach:
    • Centralized Field Names: If the field names ever change, you only need to update them in one place (FOREX_DEAL.js), and all components using those constants will automatically reflect the changes.
    • Improved Readability: Using constants makes the code easier to read and maintain because it avoids hardcoded strings scattered throughout the codebase.
    • Reusability: You can reuse the constants across multiple components without repeating the same field names, improving consistency.

    By following this approach, you can organize your constants and keep the logic of your form clean and manageable.

    • Consistency: Structured the constants and functions logically for better readability and maintainability.

    Dot Notation vs Bracket Notation

    In JavaScript, you can access object properties in two ways:

    1. Dot Notation: This is used when you know the property name in advance (e.g., object.property).
    2. Bracket Notation: This is used when you want to access a property dynamically, or when the property name contains special characters (like spaces or hyphens) or starts with a number. You use bracket notation to access properties via variables or expressions.
    const FOREX_DEAL = {
  SELLING_CURRENCY: "currencySoldId", // This is the string we want to use as a property name
};

const formValues = {
  FOREX_DEAL: {
    currencySoldId: "USD", // The value we want to access
  },
};

// This will NOT work, because `formValues?.FOREX_DEAL.SELLING_CURRENCY` is not a valid property
const sellingCurrency1 = formValues?.FOREX_DEAL.SELLING_CURRENCY;
console.log(sellingCurrency1); // undefined, because there's no property called `SELLING_CURRENCY` on `FOREX_DEAL`

// This will work, because it accesses the property `currencySoldId` dynamically using the string value stored in `FOREX_DEAL.SELLING_CURRENCY`
const sellingCurrency2 = formValues?.[FOREX_DEAL.SELLING_CURRENCY];
console.log(sellingCurrency2); // "USD", because `FOREX_DEAL.SELLING_CURRENCY` is "currencySoldId"
