    Intro

    Event Loop

    The event loop is a fundamental concept in JavaScript that enables its non-blocking, asynchronous behavior. It is the mechanism that allows JavaScript to handle tasks like I/O operations, timers, and callbacks efficiently, even though JavaScript is single-threaded.

    Quote

    The event loop in JavaScript is a mechanism and algorithm that continuously monitors the call stack and task queues, ensuring that JavaScript executes code efficiently while remaining non-blocking.

    Event Loop in Technical Terms

    The event loop works like a coordinator between:

    • Call Stack (where synchronous code runs)
    • Task Queues (where asynchronous tasks wait)
    • Web APIs (for timers, fetch requests, etc.)

    The event loop continuously checks if the call stack is empty, and if so, it pushes tasks from the task queues onto the stack for execution.

    Why is the Event Loop Important?

    JavaScript runs in a single-threaded environment, meaning it can only execute one task at a time. However, many tasks in a web application (like network requests, file reading, or timers) are asynchronous and take time to complete. The event loop ensures that these tasks don't block the main thread, allowing the application to remain responsive.

    How the Event Loop Works

    The event loop continuously monitors the call stack and the task queues to determine what to execute next. Here's a breakdown of the key components:

    1. Call Stack:

      • The call stack is where JavaScript keeps track of function calls. It operates in a Last In, First Out (LIFO) manner.
      • When a function is called, it is added to the top of the stack. When the function returns, it is removed from the stack.

    2. Web APIs:

      • Browser-provided APIs (like setTimeout, fetch, or DOM events) handle tasks that are asynchronous or take time to complete.
      • When an asynchronous operation is initiated, it is offloaded to the Web API, and the main thread continues executing other code.

    3. Callback Queue (Task Queue):

      • Once an asynchronous operation is completed, its callback function is placed in the callback queue.

      • Examples of tasks in the callback queue include:

      • Callbacks from setTimeout or setInterval.

      • Event handlers like click or keypress.
      • Callbacks from network requests (fetch or XMLHttpRequest).

    4. Microtask Queue:

      • This queue has higher priority than the callback queue and is used for promises and other microtasks.

      • Examples of microtasks include:

      • Promise.then() or Promise.catch() callbacks.

      • queueMicrotask().
      • MutationObserver callbacks.

    5. Event Loop:

      • The event loop constantly checks if the call stack is empty.
      • If the call stack is empty, it first processes all tasks in the microtask queue.
      • Once the microtask queue is empty, it processes tasks in the callback queue.

    Example of the Event Loop in Action

    console.log("Start");

setTimeout(() => {
  console.log("Timeout callback");
}, 0);

Promise.resolve().then(() => {
  console.log("Promise resolved");
});

console.log("End");
    Output
    Start
End
Promise resolved
Timeout callback
    Explanation

    1. console.log('Start') is executed and added to the call stack.

    2. setTimeout is called, and its callback is offloaded to the Web API. After the timer completes (even if it's 0ms), the callback is moved to the callback queue.

    3. Promise.resolve().then() is called, and its callback is added to the microtask queue.

    4. console.log('End') is executed and added to the call stack.

    5. The call stack is now empty, so the event loop processes the microtask queue first, logging Promise resolved.

    6. Finally, the event loop processes the callback queue, logging Timeout callback.

    Key Points About the Event Loop

    1. Single-Threaded: JavaScript has only one call stack, meaning it can only do one thing at a time.

    2. Non-Blocking: Asynchronous operations are handled by Web APIs, so they don't block the main thread.

    3. Microtasks vs Macrotasks:

      • Microtasks (e.g., promises) have higher priority and are executed before macrotasks (e.g., setTimeout).

    4. Event Loop Phases:

      • The event loop processes microtasks first, then macrotasks, and repeats this cycle.

    Visual Representation of the Event Loop

    1. Call Stack: Executes synchronous code.
    2. Web APIs: Handle asynchronous operations.
    3. Callback Queue: Holds callbacks from asynchronous operations.
    4. Microtask Queue: Holds promise callbacks and other microtasks.
    5. Event Loop: Moves tasks from the queues to the call stack when it's empty.

    Real-World Analogy

    Imagine a restaurant:

    • The call stack is the chef cooking one dish at a time.
    • The Web APIs are the kitchen assistants handling tasks like fetching ingredients or washing dishes.
    • The callback queue is the line of orders waiting to be cooked.
    • The microtask queue is a priority line for urgent orders (like a VIP customer).
    • The event loop is the manager who ensures the chef cooks orders from the priority line first, then the regular line.

    Summary

    The event loop is the backbone of JavaScript's asynchronous programming model. It ensures that asynchronous tasks are handled efficiently without blocking the main thread. By understanding the event loop, you can write more efficient and responsive JavaScript code.

    Event

    In JavaScript, an event is an action or occurrence that happens in the browser, such as a user clicking a button, resizing a window, or pressing a key. Events are a core part of interactive web applications, allowing developers to respond to user actions or other changes in the application.

    Event Basics

    1. Event Types: There are many types of events in JavaScript, such as:

      • Mouse events: click, mouseover, mouseout, mousedown, mouseup
      • Keyboard events: keydown, keyup, keypress
      • Form events: submit, change, focus, blur
      • Window events: load, resize, scroll
      • Custom events: You can also create your own custom events.

    2. Event Target: The element or object that the event occurs on. For example, if a user clicks a button, the button is the event target.

    3. Event Listener: A function that waits for a specific event to occur and then executes in response. You attach an event listener to an element using methods like addEventListener.

    4. Event Object: When an event occurs, the browser creates an event object that contains information about the event, such as the type of event, the target element, and additional data (e.g., mouse coordinates for a click event).

    Example of an Event in JavaScript

    Heres a simple example of handling a click event:

    // Select the button element
const button = document.querySelector("button");

// Add an event listener for the 'click' event
button.addEventListener("click", function (event) {
  // event is the event object
  console.log("Button clicked!");
  console.log("Event type:", event.type); // Output: "click"
  console.log("Target element:", event.target); // Output: The button element
});

    Event Flow (Bubbling and Capturing)

    Events in JavaScript follow a flow:

    1. Capturing Phase: The event starts from the top of the DOM tree and goes down to the target element.
    2. Target Phase: The event reaches the target element.
    3. Bubbling Phase: The event bubbles up from the target element back to the top of the DOM tree.

    You can control whether an event listener is triggered during the capturing or bubbling phase using the third parameter of addEventListener:

    element.addEventListener(
  "click",
  function (event) {
    console.log("Event triggered during bubbling phase");
  },
  false
); // false (default) means bubbling phase

element.addEventListener(
  "click",
  function (event) {
    console.log("Event triggered during capturing phase");
  },
  true
); // true means capturing phase

    Preventing Default Behavior

    Some events have default behaviors (e.g., a form submission or a link click). You can prevent these default behaviors using event.preventDefault():

    const link = document.querySelector("a");

link.addEventListener("click", function (event) {
  event.preventDefault(); // Prevents the link from navigating
  console.log("Link click prevented");
});

    Custom Events

    You can create and dispatch custom events using the CustomEvent constructor:

    // Create a custom event
const customEvent = new CustomEvent("myCustomEvent", {
  detail: { message: "This is a custom event" },
});

// Add an event listener for the custom event
document.addEventListener("myCustomEvent", function (event) {
  console.log("Custom event triggered:", event.detail.message);
});

// Dispatch the custom event
document.dispatchEvent(customEvent);

    Summary

    • An event in JavaScript is an action or occurrence in the browser.
    • Events are handled using event listeners attached to elements.
    • The event object contains details about the event.
    • Events flow through the DOM in three phases: capturing, target, and bubbling.
    • You can prevent default behaviors or create custom events.
