    Pure & Impure Function

    Pure Function

    A pure function in JavaScript is a function that:

    1. Always produces the same output for the same input: No matter how many times you call the function with the same arguments, the result will always be the same.
    2. Has no side effects: It does not modify external variables, data, or state.

    Key Characteristics of Pure Functions

    • Deterministic: Output depends solely on input.
    • No Side Effects: Does not interact with or modify anything outside of its scope (e.g., no I/O operations, no modifying global variables).

    Example of a Pure Function

    Example
    function add(a, b) {
  return a + b;
}

// Example usage
console.log(add(2, 3)); // Output: 5
console.log(add(2, 3)); // Output: 5 (always the same output for the same inputs)

    What Makes a Function Pure

    No Dependency on External State:

    The function must not rely on or modify variables outside of its scope.

    Impure Example:
    let x = 10;
function addToX(y) {
    return x + y; // Impure because it uses external variable x
}
    Pure Alternative:
    function add(x, y) {
    return x + y; // Pure because it only depends on input arguments
}

    No Side Effects:

    It must not modify anything outside itself (e.g., DOM, global state, etc.).

    function incrementArray(arr) {
    arr.push(arr.length); // Impure because it modifies the input array
    return arr;
}

    function getIncrementedArray(arr) {
    return [...arr, arr.length]; // Pure because it returns a new array without modifying the original
}
