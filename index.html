    Different Kinds of Loops

    JavaScript supports different kinds of loops:

    • for - loops through a block of code a number of times
    • for/in - loops through the properties of an object
    • for/of - loops through the values of an iterable object
    • while - loops through a block of code while a specified condition is true
    • do/while - also loops through a block of code while a specified condition is true

    Scopes

    In JavaScript, scope refers to the visibility of a variable or how it can be used after it is declared. The scope of a variable depends on the keyword that was used to declare it.

    The three types of Scope are Global Scope, Function Scope, and Block Scope.(1)

    1. Global Variables in HTML

      With JavaScript, the global scope is the JavaScript environment.

      In HTML, the global scope is the window object.

      Global variables defined with the var keyword belong to the window object:

      var carName = "Volvo";
// code here can use window.carName

      Global variables defined with the let keyword do not belong to the window object:

      let carName = "Volvo";
// code here can not use window.carName

    • Block scope
    • Function scope (Local Scope)

    Nullish Coalesing Operator vs Logical OR Operator

    Prefer the nullish coalescing operator (??) over the logical OR operator (||) when you want to provide a fallback value only if the original value is null or undefined, not for other falsy values like 0, "" (empty string), or false.

    JS Protocol

    The JavaScript Iteration Protocols and How They Fit In

    ES6 introduced two new protocols that work together to define how iteration should function in JavaScript: the Iterable Protocol and the Iterator Protocol. These protocols enable objects to be used with the for...of loop and other constructs that expect iterable data.

    Iterable Protocol

    The iterable protocol allows an object to define its iteration behavior. An object is considered iterable if it implements the @@iterator method, accessible via Symbol.iterator. This method must return an object that conforms to the iterator protocol.

    Iterator Protocol

    The iterator protocol defines a standard way to produce a sequence of values. An object is an iterator when it implements a next() method with the following requirements:

    The next() method must return an object with two properties:

    • value: the next value in the sequence (can be undefined if the iteration is complete).
    • done: a boolean indicating whether the iterator has completed. It's true when the sequence is finished, and false otherwise.
    Example
    const myIterable = {
  data: [1, 2, 3],
  [Symbol.iterator]() {
    let index = 0;
    return {
      next: () => {
        if (index < this.data.length) {
          return { value: this.data[index++], done: false };
        } else {
          return { value: undefined, done: true };
        }
      },
    };
  },
};

for (const value of myIterable) {
  console.log(value); // Output: 1, 2, 3
}

    In this example, myIterable is an iterable because it has a Symbol.iterator method. The method returns an iterator object with a next() method that produces values from the data array until there are no more values, at which point it signals completion.

    JavaScript Protocols: Overview and Types

    Clarifying "JavaScript Protocols"

    The term "JavaScript protocols" can refer to two distinct concepts:

    • Web/network protocols that JavaScript interacts with (like HTTP, WebSocket, etc.).
    • Native JavaScript object protocols-internal standards that objects and functions can implement to enable certain behaviors (such as iteration or observability).

    1. Native JavaScript Protocols

    JavaScript defines several protocols-standardized interfaces that objects can implement to work with built-in language features. The most notable native protocols include:

    • Iterator Protocol: Enables objects to define or customize their iteration behavior (e.g., for use in for...of loops).

    • Iterable Protocol: Allows objects to be iterated over by defining a [Symbol.iterator] method.

    • Async Iterator Protocol: Similar to the iterator protocol, but supports asynchronous iteration with [Symbol.asyncIterator].

    • Observable Protocol: A proposed protocol for creating declarative, asynchronous streams of data.

    • Generator Protocol: Functions that can be paused and resumed, producing a sequence of values.
